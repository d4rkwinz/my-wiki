# System Design

Topical hub for designing systems that scale, stay available, and degrade gracefully under load. Sub-MOCs below organize concepts by the layer or concern they belong to.

Parent: [[maps]]

## Sub-maps

- [[Networking]] — how traffic reaches the system: DNS, load balancing, edge delivery.
- [[Databases]] — persistent storage, replication, partitioning, sharding.
- [[Caching]] — fast in-memory layers and edge caches that absorb read load.
- [[Distributed Systems]] — multi-node coordination: statelessness, queues, multi-DC, failure modes.
- [[Operations]] — running the system in production: observability, deployment, redundancy.
